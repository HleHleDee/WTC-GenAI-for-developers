# Understanding FastAPI Fundamentals

**Prompt Used**

What is FastAPI and how does it compare to Flask and Django? 

Explain the core concepts and terminology used in FastAPI.

What are the key advantages of FastAPI over other Python web frameworks?

Create a glossary of essential FastAPI terms and concepts.

**AI Summary**
FastAPI is a modern Python web framework used to build APIs quickly and efficiently.

Comparison:
- Flask: lightweight but requires more manual setup
- Django: full-stack framework with more built-in features
- FastAPI: focused on APIs, fast, and supports async programming

Key Concepts:
- Path operations (routes/endpoints)
-  Pydantic models (data validation)
-  Dependency injection
-  Async support

Advantages:
- Very fast performance- Automatic API documentation (Swagger UI)
- Built-in data validation
- Easy to learn and use### 

Glossary:
- Endpoint: URL route in the API
-  Pydantic: data validation library
- Uvicorn: ASGI server used to run FastAPI
- Request body: data sent to API

**AI Summary**

FastAPI uses Pydantic models to validate incoming request data.

Key ideas:
-Models define expected data structure
- FastAPI automatically validates input
- Errors are returned automatically if invalid

Example:- 
Use BaseModel for request validation- Use routers to organize code- Use exception handlers for errors

Benefits:
- Cleaner code
- Safer APIs
- Better maintainability
