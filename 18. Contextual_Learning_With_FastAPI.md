# Contextual Learning with FastAPI

# Framework Comparison

**My Approach**

I do not have much experience with Flask or Django, so I used AI to understand how FastAPI compares to them.

**Prompt Used**

Compare FastAPI with Flask and Django. What are the similarities and differences?

**What I Learned**

-FastAPI is similar to Flask because both use simple route functions
-Django is more complex and includes many built-in features
-FastAPI automatically handles validation and documentation, unlike Flask and Django

This helped me understand where FastAPI fits among Python frameworks.


# Understanding Design Choices

**Prompts Used**

-Why does FastAPI use Pydantic?
-Why does FastAPI use type hints?
-Why is FastAPI async-first?

**What I Learned**

-FastAPI uses Pydantic to validate data automatically
-Type hints improve readability and help generate documentation
-Async allows FastAPI to handle multiple requests efficiently

**Summary**

FastAPI is designed to reduce repetitive code and make API development faster and cleaner.


# Applied Contextual Learning (JWT Authentication)

**What I Did**

I used the provided FastAPI JWT example and asked AI to explain how it works step by step.

**Prompt Used**

Explain how JWT authentication works in this FastAPI code step by step.

**What I Learned**

-A user logs in and receives a token
-The token is used to access protected routes
-FastAPI uses Depends() to check authentication
-The token includes user information and expiration time

**My Understanding**

Even without prior experience in other frameworks, this helped me understand how authentication works in APIs and how FastAPI simplifies the process.
